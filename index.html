Form e suas configurações

Ele serve para criar formularios em HTML, o usúario preenche as informações e manda elas para outro lugar.

  • Ele é escrito como uma tag <form action="/cadastro" method="post">
    1. ACTION
      • Documentação de explicação do que é action
      • Resumidamente, é o caminho para onde os dados vai depois do submit.
    2. METHOD
      • Documentação de explicação do que é method
      • Resumidamente, é o tipo de requisição, de qual forma os dados serão enviados depois do submit.
        1. GET
          • Get é o metodo usado para enviar os dados pela URL, eel ficara visivel na URL, nao é bom colocar para quand for pesquisa grande, pois tem limites de caracteres, 3000. não pode se usar para arquivos "secretos"
        2. POST
          • Post é o metodo usado para enviar os dados de forma oculta, não ficara visivel na URL. é o metodo mais seguro para enviar dados sensíveis.
    3. TIPOS DE CAMPOS
      • Documentação de explicação dos tipos de campos
      • Resumidamente, são os diferentes tipos de entrada que podem ser usados em um formulário.
        1. BUTTON
          • Mais usado para JS, ativar alert e outras coisas, ele nao envia o formulario
        2. CHECKBOX
          • Usado para criar caixa de seleção de multiplas escolhas, deve ter o mesmo "name=", o que deve nudar e é "value="
        3. COLOR
          • usado para o usuario selecionar uma cor
        4. DATE
          • usado para o usuario selecionar uma data
        5. DATETIME-local
          • usado para o usuario selecionar uma data e hora
        6. EMAIL
          • usado para o usuario inserir um endereço de email, ja com validação - precisa conter o @ para ser valido
        7. FILE
          • usado para o usuario selecionar um arquivo do computador para enviar
        8. HIDDEN
          • usado para enviar dados ocultos, não é visivel para o usuario, mas é enviado junto com o formulario
        9. IMAGE
          • É um botão de envio (submit) que usa uma imagem em vez de texto. Funciona como um botão de submit, mas com uma imagem como aparência.
        10. MONTH
          • O usuário seleciona o mês e o ano
        11. NUMBER
          • usado para o usuario inserir um numero
        12. PASSWORD
          • O usuário coloca uma senha
        13. RADIO
          • Usado para criar opções de escolha única, deve ter o mesmo "name=", o que deve nudar e é "value="
        14. RANGE
          • O usuário seleciona um valor dentro de um intervalo, geralmente exibido como um controle deslizante (slider).
        15. RESET
          • O reset serve para resetar os valores do formulario
        16. SEARCH
          • O usuário insere um termo de pesquisa, geralmente exibido com um campo de texto e um botão de pesquisa. MUito usado em motores de busca
        17. SUBMIT
          • O submit é o botão de envio do formulario, ele envia os dados para o servidor ou para a URL especificada no atributo "action" do formulário.
        18. TEL
          • O usuário coloca um número de telefone
        19. TIME
          • O usuário seleciona um horário
        20. URL
          • O usuário insere uma URL
        21. WEEK
          • O usuário seleciona uma semana
        22. TEXT
          • O usuário insere um texto
    4. TEXTAREA
      • O usuário insere um texto de múltiplas linhas
    5. SELECT
      • O usuário seleciona uma opção de uma lista de itens, o usuario nao digita, apenas escolhe
        1. SIZE
          • Deixa a lista com um número específico de opções visíveis
    6. OPTION
      • Representa uma única opção dentro de um elemento SELECT
        • SELECTED
          • Indica que a opção é a opção selecionada por padrão quando a página é carregada, já deixa ela de padrão de escolha para o usuário
        • DISABLED
          • Indica que a opção está desativada e não pode ser selecionada
    7. OPTGROUP
      • Representa um grupo de opções dentro de um elemento SELECT
